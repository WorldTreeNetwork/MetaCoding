|---|---|---|---|---|

| `createPlantingPlan` | mutator | no | Appends a planting-plan creation event; only a nonempty seasons selection updates this actor's remembered season defaults, while an empty selection leaves the prior defaults unchanged. | invented: remembered selected seasons is conditional state written by the plan-creation workflow, not a single source method |

| `confirmRecord` | mutator | no | Appends a confirmation status event for a previously pending record; later status events supersede earlier status in status-sensitive projections. | invented: pending-versus-done lifecycle status is produced by source form workflows rather than a Harvest class method |

## Rationale

Mutators append domain events and return opaque handles; all handles use the fixed replica-scoped client id (prefix_replicaId~counter), collision-free by construction, no merge step needed. Planting is composite so asset creation and its movement taxonomy replay together: seeding/transplanting are distinct movement kinds with isLog:false, whereas harvest is a log with isMovement:false. The status-sensitive projections implement the frozen pending-status contract relevant here: location requires confirmed, while yieldTotal/logCount require confirmed with pending-only partners. Actor-bearing projections are disclosure-filtered materialized views: access is re-answered at the disclosure boundary rather than through an authoritative request-time write gate. Birth resolution uses the bound uniqueness rule verbatim, and revision history intentionally admits concurrent append-only histories.
